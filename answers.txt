1. Обязательно ли скачивать и устанавливать какие-то библиотеки для начала работы с ReactJS?

> Самый простой способ создания проекта с ReactJS — с помощью npm команды “create-react-app”, которая создаёт определенную структуру проекта. (Для этого нужен установленный Node.js и npm). Но если нужно подключить React к существующему проекту, тогда через тег <script> (Как jQuery). Формально, при подключении React’a в проект подгружаются  файлы с сервера, где лежит библиотека, но глобально на ПК ничего не устанавливается.


2. Плюсы и минусы приложений с SPA.

> Плюсы: быстрая работа приложения, за счёт асинхронной работы скрипта, независимая работа компонентов, экономия памяти сервера и интернет-трафика, простота использования и создания. Минусы: плохая SEO оптимизация, из-за тяжелых фреймворков нагружают браузер, не у всех работает JavaScript, не подходит для сильно сложных приложений и в целом разработка стоит больших денег.


3. Как создать React-приложение? 

> Через “create-react-app” — подходит для SPA-приложений.


4. Является ли React фреймворком? И какие фреймворки есть ещё в веб-фронтенде. 

> React по сути не фреймворк, а бибилиотека с готовыми решениями различных частных задач, а не проекта в целом. Фреймворк подрузамевает под собой создание определенного «фрейма» вокруг проекта. Но, зачастую, React используют в связке с другими библиотеками и фреймворками (Redux, Relay). Во веб-фронтенде существуют различные фреймворки — Angular.js, Ember.js, Express.js, Next.js, Vue.js.
 

5. Какой командой можно добавить библиотеку или компонент в свой файл?

7. Какой основной файл SPA-проекта, где мы делаем изменения?

8. Зачем нужна папка build и какой командой мы генерируем ее содержимое? 

> build содержит в себе файлы для продакшена, то есть для загрузки на сервер. генерируется с помощью команды npm build.
